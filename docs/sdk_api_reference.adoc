== Abstract

This document provides a comprehensive reference for the RV32 SoC Software Development Kit (SDK).
It describes the API functions, register definitions, and programming model for all peripherals
and system components in the RV32 RISC-V System-on-Chip.

The SDK provides a header-only C library optimized for bare-metal embedded applications,
with zero overhead when functions are not used.

'''

== Introduction

=== Document Scope

This API reference covers:

* Memory-mapped I/O (MMIO) peripheral drivers
* Interrupt controller configuration (PLIC, CLINT)
* Control and Status Register (CSR) access
* System initialization and exception handling
* Debug and simulation utilities

=== SDK Architecture

The RV32 SDK follows these design principles:

Header-Only Design:: All driver functions are implemented as `static inline` functions,
eliminating the need to link against a separate library. Unused functions contribute zero code size.

Zero Overhead Abstraction:: Direct register access with no additional memory or CPU overhead compared to raw MMIO operations.

Type Safety:: All register addresses and bit fields are defined as compile-time constants with proper types.

Portability:: Works with both RTL simulation and software simulators (rv32sim, Spike).

=== Header Organization

[cols="2,3,5"]
|===
| Header File | Peripheral/Module | Description

| `rv_platform.h`
| Platform definitions
| Base addresses, register offsets, bit-field constants for all peripherals

| `rv_uart.h`
| UART
| Serial communication driver (polling and interrupt modes)

| `rv_spi.h`
| SPI
| SPI master controller driver

| `rv_i2c.h`
| I2C
| I2C master controller driver

| `rv_plic.h`
| PLIC
| Platform-Level Interrupt Controller configuration

| `rv_clint.h`
| CLINT
| Core-Local Interrupt Controller (timer and software interrupts)

| `rv_irq.h`
| Interrupt Management
| Global and per-source interrupt control, handler registration

| `csr.h`
| CSR Access
| Control and Status Register read/write functions
|===

=== Memory-Mapped Register Access

All peripherals use 32-bit aligned registers. The SDK provides the `RV_REG32` macro for safe MMIO access:

[source,c]
----
#define RV_REG32(base, off) \
    (*(volatile uint32_t *)((uintptr_t)(base) + (uintptr_t)(off)))
----

Example usage:

[source,c]
----
uint32_t status = RV_REG32(RV_UART_BASE, RV_UART_STATUS_OFF);
RV_REG32(RV_UART_BASE, RV_UART_DATA_OFF) = 0x41; // Write 'A'
----

== UART Driver API

=== Overview

The UART peripheral (`axi_uart.sv`) provides full-duplex serial communication with:

* Configurable baud rate
* 8-bit data, no parity, 1 stop bit (8N1 format)
* TX and RX FIFOs (16 entries each)
* Interrupt generation on RX data available and TX FIFO empty
* Internal loopback mode for testing

Base Address:: `0x2000_0000` (RV_UART_BASE)
PLIC Source ID:: 1 (RV_PLIC_SRC_UART)

=== Register Map

[cols="2,2,2,6"]
|===
| Offset | Name | Access | Description

| `0x00`
| DATA
| R/W
| TX (write) / RX (read) data register

| `0x04`
| STATUS
| RO
| Status flags (TX busy, RX ready, FIFO full)

| `0x08`
| IE
| R/W
| Interrupt Enable register

| `0x0C`
| IS
| R/W1C
| Interrupt Status register (write-1-to-clear)

| `0x10`
| LEVEL
| R/W
| Baud rate divisor

| `0x14`
| CTRL
| R/W
| Control register (loopback enable)
|===

=== Status Bits

[cols="1,2,5"]
|===
| Bit | Name | Description

| 0
| TX_BUSY / TX_FULL
| TX FIFO full – cannot write

| 2
| RX_READY
| RX FIFO not empty – data available

| 3
| RX_FULL
| RX FIFO full
|===

=== API Functions

==== Initialization

[source,c]
----
void rv_uart_init(uint32_t baud_div);
----

Initialize the UART peripheral with the specified baud rate divisor.

*Parameters:*

* `baud_div`: Baud rate divisor = `(sys_clk_hz / (baud_rate * 8)) - 1`

*Example:*

[source,c]
----
// Configure for 115200 baud at 100 MHz system clock
// baud_div = (100000000 / (115200 * 8)) - 1 = 107
rv_uart_init(107);
----

==== Status Queries

[source,c]
----
int rv_uart_tx_busy(void);    // Returns 1 if TX FIFO full
int rv_uart_rx_ready(void);   // Returns 1 if RX data available
----

==== Polling Transmit

[source,c]
----
void rv_uart_putc(char c);
void rv_uart_puts(const char *s);
void rv_uart_write(const uint8_t *buf, uint32_t len);
----

*Description:*

* `rv_uart_putc()`: Block until TX FIFO has space, then transmit one character
* `rv_uart_puts()`: Transmit a null-terminated string
* `rv_uart_write()`: Transmit len bytes from buffer

*Example:*

[source,c]
----
rv_uart_puts("Hello, World!\n");
----

==== Polling Receive

[source,c]
----
int rv_uart_getc(void);              // Non-blocking, returns -1 if empty
uint8_t rv_uart_getc_blocking(void); // Blocks until data available
----

*Example:*

[source,c]
----
int c;
while ((c = rv_uart_getc()) != -1) {
    rv_uart_putc((char)c);  // Echo received character
}
----

==== Interrupt Control

[source,c]
----
void rv_uart_irq_enable(uint32_t mask);
void rv_uart_irq_disable(uint32_t mask);
uint32_t rv_uart_irq_status(void);
----

*Interrupt Sources:*

* `RV_UART_IE_RX_READY` (bit 0): RX FIFO not empty
* `RV_UART_IE_TX_EMPTY` (bit 1): TX FIFO drained (empty)

*Example:*

[source,c]
----
// Enable RX interrupt
rv_uart_irq_enable(RV_UART_IE_RX_READY);

// In interrupt handler:
uint32_t status = rv_uart_irq_status();
if (status & RV_UART_IE_RX_READY) {
    char c = rv_uart_getc();
    // Process received character
}
----

==== Loopback Mode

[source,c]
----
void rv_uart_loopback_enable(void);
void rv_uart_loopback_disable(void);
----

Enable internal TX→RX loopback for hardware testing. Transmitted bytes are fed back to the RX path without leaving the chip.

'''

== SPI Driver API

=== Overview

The SPI peripheral (`axi_spi.sv`) provides a master-mode SPI controller with:

* Configurable clock frequency
* All four SPI modes (CPOL, CPHA)
* Up to 4 independent chip-select lines
* TX and RX FIFOs (16 entries each)
* Full-duplex operation
* Internal loopback mode

Base Address:: `0x2002_0000` (RV_SPI_BASE)
PLIC Source ID:: 2 (RV_PLIC_SRC_SPI)

=== Register Map

[cols="2,2,2,6"]
|===
| Offset | Name | Access | Description

| `0x00`
| CTRL
| R/W
| Control register (enable, CPOL, CPHA, CS, loopback)

| `0x04`
| DIV
| R/W
| Clock divisor

| `0x08`
| TX
| WO
| TX FIFO push

| `0x0C`
| RX
| RO
| RX FIFO pop

| `0x10`
| STATUS
| RO
| Status flags

| `0x14`
| IE
| R/W
| Interrupt Enable

| `0x18`
| IS
| R/W1C
| Interrupt Status
|===

=== SPI Modes

[cols="1,1,1,4"]
|===
| Mode | CPOL | CPHA | Description

| MODE0
| 0
| Clock idle low, sample on rising edge

| MODE1
| 0
| 1
| Clock idle low, sample on falling edge

| MODE2
| 1
| 0
| Clock idle high, sample on falling edge

| MODE3
| 1
| Clock idle high, sample on rising edge
|===

=== API Functions

==== Initialization

[source,c]
----
void rv_spi_init(uint32_t clk_div, uint32_t mode);
----

*Parameters:*

* `clk_div`: SCK frequency = `sys_clk / (2 * (clk_div + 1))`
* `mode`: `RV_SPI_MODE0`, `MODE1`, `MODE2`, or `MODE3`

*Example:*

[source,c]
----
// Configure for 1 MHz SPI clock at 100 MHz system clock, mode 0
// clk_div = (100000000 / (2 * 1000000)) - 1 = 49
rv_spi_init(49, RV_SPI_MODE0);
----

==== Chip Select Control

[source,c]
----
void rv_spi_cs_select(uint32_t n);   // Assert CS[n] (0-3)
void rv_spi_cs_deselect(void);       // Deassert all CS lines
----

*Example:*

[source,c]
----
rv_spi_cs_select(0);        // Assert CS0
// Perform transfer
rv_spi_cs_deselect();       // Release CS0
----

==== Data Transfer

[source,c]
----
uint8_t rv_spi_transfer(uint8_t data);
void rv_spi_transfer_buf(const uint8_t *tx_buf, uint8_t *rx_buf, uint32_t len);
void rv_spi_read_buf(uint8_t *rx_buf, uint32_t len);
----

*Description:*

* `rv_spi_transfer()`: Full-duplex transfer of one byte
* `rv_spi_transfer_buf()`: Transfer multiple bytes (tx_buf or rx_buf may be NULL)
* `rv_spi_read_buf()`: Read len bytes by clocking out 0xFF

*Example:*

[source,c]
----
uint8_t tx_data[] = {0x03, 0x00, 0x00, 0x00}; // Read command + address
uint8_t rx_data[256];

rv_spi_cs_select(0);
rv_spi_transfer_buf(tx_data, NULL, 4);  // Send command
rv_spi_read_buf(rx_data, 256);          // Read data
rv_spi_cs_deselect();
----

==== Status and Control

[source,c]
----
int rv_spi_busy(void);        // Transfer in progress
int rv_spi_tx_ready(void);    // Can accept TX data
int rv_spi_rx_valid(void);    // RX data available
void rv_spi_wait_ready(void); // Block until idle
----

==== Interrupt Control

[source,c]
----
void rv_spi_irq_enable(uint32_t mask);
void rv_spi_irq_disable(uint32_t mask);
uint32_t rv_spi_irq_status(void);
----

*Interrupt Sources:*

* `RV_SPI_IE_RX_READY` (bit 0): RX FIFO not empty
* `RV_SPI_IE_TX_EMPTY` (bit 1): TX FIFO empty

==== Loopback Mode

[source,c]
----
void rv_spi_loopback_enable(void);
void rv_spi_loopback_disable(void);
----

'''

== I2C Driver API

=== Overview

The I2C peripheral (`axi_i2c.sv`) provides a master-mode I2C controller with:

* Configurable clock frequency (100 kHz / 400 kHz / custom)
* 7-bit addressing
* START/STOP condition generation
* ACK/NACK detection
* Interrupt support

Base Address:: `0x2001_0000` (RV_I2C_BASE)
PLIC Source ID:: 3 (RV_PLIC_SRC_I2C)

=== Register Map

[cols="2,2,2,6"]
|===
| Offset | Name | Access | Description

| `0x00`
| CTRL
| R/W
| Control register (enable, START, STOP, READ, NACK)

| `0x04`
| DIV
| R/W
| Clock divisor

| `0x08`
| TX
| WO
| Transmit data register

| `0x0C`
| RX
| RO
| Receive data register

| `0x10`
| STATUS
| RO
| Status flags (busy, ACK received, RX valid)

| `0x14`
| IE
| R/W
| Interrupt Enable

| `0x18`
| IS
| R/W1C
| Interrupt Status
|===

=== API Functions

==== Initialization

[source,c]
----
void rv_i2c_init(uint32_t clk_div);
----

*Parameters:*

* `clk_div`: SCL period = `sys_clk / (4 * (clk_div + 1))`

*Example:*

[source,c]
----
// Configure for 100 kHz at 100 MHz system clock
// clk_div = (100000000 / (4 * 100000)) - 1 = 249
rv_i2c_init(249);

// Configure for 400 kHz at 100 MHz system clock
// clk_div = (100000000 / (4 * 400000)) - 1 = 62
rv_i2c_init(62);
----

==== Low-Level Operations

[source,c]
----
void rv_i2c_start(void);                    // Issue START condition
void rv_i2c_stop(void);                     // Issue STOP condition
int rv_i2c_write_byte(uint8_t data);        // Write byte, return 0 if ACK
uint8_t rv_i2c_read_byte(int send_ack);     // Read byte, send ACK if 1
----

==== High-Level Transfers

[source,c]
----
int rv_i2c_master_write(uint8_t addr, const uint8_t *buf, uint32_t len);
int rv_i2c_master_read(uint8_t addr, uint8_t *buf, uint32_t len);
----

*Description:*

Complete write or read transactions including START, address, data, and STOP.

*Parameters:*

* `addr`: 7-bit I2C device address
* `buf`: Data buffer
* `len`: Number of bytes to transfer

*Return Values:*

* `0`: Success
* `-1`: Device did not ACK the address
* `-2`: Device did not ACK a data byte

*Example:*

[source,c]
----
// Write to EEPROM at address 0x50
uint8_t eeprom_addr = 0x50;
uint8_t write_data[] = {0x00, 0x10, 0xAA, 0xBB}; // Addr MSB, LSB, data

if (rv_i2c_master_write(eeprom_addr, write_data, 4) < 0) {
    // Handle error
}

// Read from EEPROM
uint8_t read_buf[2];
if (rv_i2c_master_read(eeprom_addr, read_buf, 2) < 0) {
    // Handle error
}
----

==== Status and Control

[source,c]
----
int rv_i2c_busy(void);           // Transfer in progress
int rv_i2c_ack_received(void);   // Slave ACKed last byte
int rv_i2c_rx_valid(void);       // RX data available
void rv_i2c_wait_ready(void);    // Block until idle
----

==== Interrupt Control

[source,c]
----
void rv_i2c_irq_enable(uint32_t mask);
void rv_i2c_irq_disable(uint32_t mask);
uint32_t rv_i2c_irq_status(void);
----

*Interrupt Sources:*

* `RV_I2C_IE_RX_READY` (bit 0): RX FIFO not empty
* `RV_I2C_IE_TX_EMPTY` (bit 1): TX FIFO empty
* `RV_I2C_IE_STOP_DONE` (bit 2): STOP condition completed

'''

== DMA Controller API

=== Overview

The DMA controller (`axi_dma.sv`) provides high-performance data transfers with:

* 8 independent channels
* Multiple transfer modes: 1D, 2D strided, 3D planar, scatter-gather
* Configurable source/destination address increment
* Interrupt generation on completion or error
* Performance counters for throughput monitoring

Base Address:: `0x2003_0000` (RV_DMA_BASE)
PLIC Source ID:: 4 (RV_PLIC_SRC_DMA)

=== Register Map

==== Per-Channel Registers

Each channel has registers at `RV_DMA_BASE + channel * 0x40`:

[cols="2,2,2,6"]
|===
| Offset | Name | Access | Description

| `0x00`
| CTRL
| R/W
| Control register

| `0x04`
| STAT
| R/W1C
| Status register

| `0x08`
| SRC
| R/W
| Source address

| `0x0C`
| DST
| R/W
| Destination address

| `0x10`
| XFER
| R/W
| Transfer size (bytes)

| `0x14`
| SSTRIDE
| R/W
| Source stride (2D mode)

| `0x18`
| DSTRIDE
| R/W
| Destination stride (2D mode)

| `0x1C`
| ROWCNT
| R/W
| Row count (2D/3D mode)

| `0x20`
| SPSTRIDE
| R/W
| Source plane stride (3D mode)

| `0x24`
| DPSTRIDE
| R/W
| Destination plane stride (3D mode)

| `0x28`
| PLANECNT
| R/W
| Plane count (3D mode)

| `0x2C`
| SGADDR
| R/W
| Scatter-gather descriptor address

| `0x30`
| SGCNT
| R/W
| Scatter-gather descriptor count
|===

==== Global Registers

[cols="2,2,2,6"]
|===
| Offset | Name | Access | Description

| `0xF00`
| IRQ_STAT
| R/W1C
| Channel done/error flags (one bit per channel)

| `0xF04`
| IRQ_EN
| R/W
| Global interrupt enable per channel

| `0xF08`
| ID
| RO
| Identification register (0xD4A00100)

| `0xF10`
| PERF_CTRL
| R/W
| Performance counter control

| `0xF14`
| PERF_CYCLES
| R/W
| Cycles elapsed

| `0xF18`
| PERF_RD_BYTES
| R/W
| Total bytes read

| `0xF1C`
| PERF_WR_BYTES
| R/W
| Total bytes written
|===

=== Control Register Bits

[cols="1,2,5"]
|===
| Bit(s) | Name | Description

| 0
| EN
| Enable channel

| 1
| START
| Arm transfer (auto-clears)

| 2
| STOP
| Abort transfer (auto-clears)

| 4:3
| MODE
| `00`=1D, `01`=2D, `10`=3D, `11`=Scatter-Gather

| 5
| SRC_INC
| Increment source address

| 6
| DST_INC
| Increment destination address

| 7
| IE
| Interrupt enable for this channel
|===

=== Status Register Bits

[cols="1,2,5"]
|===
| Bit | Name | Description

| 0
| BUSY
| Transfer in progress (read-only)

| 1
| DONE
| Transfer complete (write-1-to-clear)

| 2
| ERR
| AXI error occurred (write-1-to-clear)
|===

=== Transfer Modes

==== 1D Transfer Mode

Simple linear memory copy:

[source,c]
----
// Copy 1024 bytes from src to dst
RV_DMA_CH_REG(0, RV_DMA_CH_SRC_OFF) = src_addr;
RV_DMA_CH_REG(0, RV_DMA_CH_DST_OFF) = dst_addr;
RV_DMA_CH_REG(0, RV_DMA_CH_XFER_OFF) = 1024;
RV_DMA_CH_REG(0, RV_DMA_CH_CTRL_OFF) =
    RV_DMA_CTRL_EN | RV_DMA_CTRL_MODE_1D |
    RV_DMA_CTRL_SRC_INC | RV_DMA_CTRL_DST_INC | RV_DMA_CTRL_START;

// Wait for completion
while (RV_DMA_CH_REG(0, RV_DMA_CH_STAT_OFF) & RV_DMA_STAT_BUSY) {}
----

==== 2D Strided Transfer Mode

Copy rectangular regions with different pitches:

[source,c]
----
// Copy 640x480 image with padding
// Source: 640 bytes/row, stride 1024
// Dest:   640 bytes/row, stride 800
RV_DMA_CH_REG(0, RV_DMA_CH_SRC_OFF) = src_addr;
RV_DMA_CH_REG(0, RV_DMA_CH_DST_OFF) = dst_addr;
RV_DMA_CH_REG(0, RV_DMA_CH_XFER_OFF) = 640;        // Bytes per row
RV_DMA_CH_REG(0, RV_DMA_CH_SSTRIDE_OFF) = 1024;    // Source row stride
RV_DMA_CH_REG(0, RV_DMA_CH_DSTRIDE_OFF) = 800;     // Dest row stride
RV_DMA_CH_REG(0, RV_DMA_CH_ROWCNT_OFF) = 480;      // Number of rows
RV_DMA_CH_REG(0, RV_DMA_CH_CTRL_OFF) =
    RV_DMA_CTRL_EN | RV_DMA_CTRL_MODE_2D |
    RV_DMA_CTRL_SRC_INC | RV_DMA_CTRL_DST_INC | RV_DMA_CTRL_START;
----

==== 3D Planar Transfer Mode

Copy 3D data structures (e.g., video frames):

[source,c]
----
// Copy video buffer: 640x480x3 (RGB planes)
RV_DMA_CH_REG(0, RV_DMA_CH_SRC_OFF) = src_addr;
RV_DMA_CH_REG(0, RV_DMA_CH_DST_OFF) = dst_addr;
RV_DMA_CH_REG(0, RV_DMA_CH_XFER_OFF) = 640;           // Bytes per row
RV_DMA_CH_REG(0, RV_DMA_CH_SSTRIDE_OFF) = 640;        // Source row stride
RV_DMA_CH_REG(0, RV_DMA_CH_DSTRIDE_OFF) = 640;        // Dest row stride
RV_DMA_CH_REG(0, RV_DMA_CH_ROWCNT_OFF) = 480;         // Rows per plane
RV_DMA_CH_REG(0, RV_DMA_CH_SPSTRIDE_OFF) = 307200;    // Source plane stride
RV_DMA_CH_REG(0, RV_DMA_CH_DPSTRIDE_OFF) = 307200;    // Dest plane stride
RV_DMA_CH_REG(0, RV_DMA_CH_PLANECNT_OFF) = 3;         // Number of planes
RV_DMA_CH_REG(0, RV_DMA_CH_CTRL_OFF) =
    RV_DMA_CTRL_EN | RV_DMA_CTRL_MODE_3D |
    RV_DMA_CTRL_SRC_INC | RV_DMA_CTRL_DST_INC | RV_DMA_CTRL_START;
----

=== Performance Monitoring

[source,c]
----
// Enable performance counters
RV_DMA_GLB_REG(RV_DMA_PERF_CTRL_OFF) = 0x3; // Reset + enable

// Start DMA transfer
// ...

// Wait for completion
// ...

// Read statistics
uint32_t cycles = RV_DMA_GLB_REG(RV_DMA_PERF_CYCLES_OFF);
uint32_t rd_bytes = RV_DMA_GLB_REG(RV_DMA_PERF_RD_BYTES_OFF);
uint32_t wr_bytes = RV_DMA_GLB_REG(RV_DMA_PERF_WR_BYTES_OFF);

// Calculate throughput
float mbps = (float)(rd_bytes + wr_bytes) / cycles * sys_clk_hz / 1e6;
----

== Platform-Level Interrupt Controller (PLIC)

=== Overview

The PLIC (`axi_plic.sv`) manages external interrupt sources from peripherals.

Base Address:: `0x0C00_0000` (RV_PLIC_BASE)

=== Interrupt Sources

[cols="1,2,4"]
|===
| ID | Source | Description

| 1
| UART
| Serial port RX ready or TX empty

| 2
| SPI
| SPI RX ready or TX empty

| 3
| I2C
| I2C RX ready, TX empty, or STOP done

| 4
| DMA
| DMA channel done or error
|===

=== API Functions

==== Source Control

[source,c]
----
void rv_plic_set_priority(uint32_t src, uint32_t priority);
void rv_plic_enable_source(uint32_t src);
void rv_plic_disable_source(uint32_t src);
int rv_plic_is_pending(uint32_t src);
void rv_plic_set_threshold(uint32_t threshold);
----

*Parameters:*

* `src`: Source ID (1-31)
* `priority`: Priority level (0-7, where 0 = disabled, 7 = highest)
* `threshold`: Only sources with priority > threshold are forwarded

==== Interrupt Handling

[source,c]
----
uint32_t rv_plic_claim(void);
void rv_plic_complete(uint32_t src);
----

*Description:*

* `rv_plic_claim()`: Read the highest-priority pending interrupt source (returns 0 if none)
* `rv_plic_complete()`: Signal completion of interrupt handling

==== Initialization Helper

[source,c]
----
void rv_plic_init_source(uint32_t src, uint32_t priority);
----

Convenience function that sets priority, enables source, sets threshold to 0, and enables external interrupts in `mie`.

*Example:*

[source,c]
----
// Initialize PLIC for UART interrupts
rv_plic_init_source(RV_PLIC_SRC_UART, 5);  // Priority 5

// Enable UART RX interrupt
rv_uart_irq_enable(RV_UART_IE_RX_READY);

// Enable global interrupts
rv_irq_enable();

// Interrupt handler
void external_irq_handler(void) {
    uint32_t src = rv_plic_claim();

    if (src == RV_PLIC_SRC_UART) {
        uint32_t status = rv_uart_irq_status();
        if (status & RV_UART_IE_RX_READY) {
            char c = rv_uart_getc();
            // Process character
        }
    }

    rv_plic_complete(src);
}
----

'''

== Core-Local Interrupt Controller (CLINT)

=== Overview

The CLINT (`axi_clint.sv`) provides timer and software interrupts.

Base Address:: `0x0200_0000` (RV_CLINT_BASE)

=== Features

* 64-bit free-running timer (`mtime`)
* 64-bit timer compare register (`mtimecmp`)
* Machine software interrupt (MSIP)

=== API Functions

==== Timer Operations

== Magic Device (Simulation/Test)

=== Overview

The Magic Device provides a simple interface for firmware to communicate with simulation environments without requiring a full UART implementation.

Base Address:: `0xFFFF_0000` (RV_MAGIC_BASE)

NOTE: This peripheral is only available in simulation (RTL testbench, rv32sim, Spike plugins).
It will not be synthesized for production FPGA/ASIC.

=== Registers

[cols="2,2,6"]
|===
| Offset | Name | Description

| `0xFFF0`
| EXIT
| Write exit code to signal simulation end

| `0xFFF4`
| CONSOLE
| Write character (low byte) to simulation console
|===

=== Exit Code Encoding

The EXIT register uses HTIF tohost encoding:

* Success (code 0): Write `1`
* Failure (code N): Write `(N << 1) \| 1`

=== API Functions

[source,c]
----
void rv_magic_putc(char c);
void rv_magic_exit(int code);
----

*Description:*

* `rv_magic_putc()`: Output one character to simulation console
* `rv_magic_exit()`: Signal program exit and halt (infinite loop)

*Example:*

[source,c]
----
#include "rv_platform.h"

int main(void) {
    const char *msg = "Hello from RV32!\n";

    // Output string
    for (const char *p = msg; *p; p++) {
        rv_magic_putc(*p);
    }

    // Signal successful completion
    rv_magic_exit(0);  // Simulation will stop here

    return 0;
}

== Complete Examples

=== Example 1: UART Echo with Interrupts

[source,c]
----
#include "rv_platform.h"
#include "rv_uart.h"
#include "rv_plic.h"
#include "rv_irq.h"

#define UART_BAUD_DIV 107  // 115200 baud at 100 MHz

volatile char rx_buffer[256];
volatile uint8_t rx_head = 0;
volatile uint8_t rx_tail = 0;

void external_irq_handler(void) {
    uint32_t src = rv_plic_claim();

    if (src == RV_PLIC_SRC_UART) {
        uint32_t status = rv_uart_irq_status();

        if (status & RV_UART_IE_RX_READY) {
            // Read all available characters
            while (rv_uart_rx_ready()) {
                rx_buffer[rx_head++] = rv_uart_getc();
            }
        }
    }

    rv_plic_complete(src);
}

int main(void) {
    // Initialize UART
    rv_uart_init(UART_BAUD_DIV);
    rv_uart_puts("UART Echo Demo\n");

    // Configure PLIC and enable interrupts
    rv_plic_init_source(RV_PLIC_SRC_UART, 5);
    rv_uart_irq_enable(RV_UART_IE_RX_READY);
    rv_irq_enable();

    // Main loop: echo received characters
    while (1) {
        if (rx_head != rx_tail) {
            char c = rx_buffer[rx_tail++];
            rv_uart_putc(c);
        }
    }

    return 0;
}

=== Example 3: I2C EEPROM Access

[source,c]
----
#include "rv_platform.h"
#include "rv_i2c.h"

#define I2C_CLK_DIV 249  // 100 kHz at 100 MHz
#define EEPROM_ADDR 0x50

int eeprom_write(uint16_t mem_addr, const uint8_t *data, uint8_t len) {
    uint8_t buf[66];  // addr(2) + data(64 max)

    if (len > 64) return -1;

    buf[0] = (mem_addr >> 8) & 0xFF;
    buf[1] = mem_addr & 0xFF;
    for (uint8_t i = 0; i < len; i++) {
        buf[i + 2] = data[i];
    }

    return rv_i2c_master_write(EEPROM_ADDR, buf, len + 2);
}

int eeprom_read(uint16_t mem_addr, uint8_t *data, uint8_t len) {
    uint8_t addr_buf[2];
    addr_buf[0] = (mem_addr >> 8) & 0xFF;
    addr_buf[1] = mem_addr & 0xFF;

    // Write address
    rv_i2c_start();
    if (rv_i2c_write_byte((EEPROM_ADDR << 1) | 0) < 0) {
        rv_i2c_stop();
        return -1;
    }
    rv_i2c_write_byte(addr_buf[0]);
    rv_i2c_write_byte(addr_buf[1]);

    // Read data
    rv_i2c_start();  // Repeated START
    if (rv_i2c_write_byte((EEPROM_ADDR << 1) | 1) < 0) {
        rv_i2c_stop();
        return -1;
    }
    for (uint8_t i = 0; i < len; i++) {
        data[i] = rv_i2c_read_byte(i < len - 1);
    }
    rv_i2c_stop();

    return 0;
}

int main(void) {
    uint8_t write_data[] = {0xAA, 0xBB, 0xCC, 0xDD};
    uint8_t read_data[4];

    rv_i2c_init(I2C_CLK_DIV);

    // Write to EEPROM
    if (eeprom_write(0x0100, write_data, 4) < 0) {
        rv_magic_exit(1);
    }

    // Small delay for write cycle
    for (volatile int i = 0; i < 10000; i++);

    // Read back
    if (eeprom_read(0x0100, read_data, 4) < 0) {
        rv_magic_exit(2);
    }

    // Verify
    for (int i = 0; i < 4; i++) {
        if (read_data[i] != write_data[i]) {
            rv_magic_exit(3);
        }
    }

    rv_magic_exit(0);
    return 0;
}
----

=== Example 4: DMA Memory Copy with Performance Monitoring

[source,c]
----
#include "rv_platform.h"

#define DMA_CHANNEL 0
#define DATA_SIZE 4096

__attribute__((aligned(4))) uint8_t src_buffer[DATA_SIZE];
__attribute__((aligned(4))) uint8_t dst_buffer[DATA_SIZE];

void dma_memcpy(void *dst, const void *src, uint32_t len) {
    // Configure DMA channel 0 for 1D transfer
    RV_DMA_CH_REG(DMA_CHANNEL, RV_DMA_CH_SRC_OFF) = (uint32_t)src;
    RV_DMA_CH_REG(DMA_CHANNEL, RV_DMA_CH_DST_OFF) = (uint32_t)dst;
    RV_DMA_CH_REG(DMA_CHANNEL, RV_DMA_CH_XFER_OFF) = len;

    // Reset and enable performance counters
    RV_DMA_GLB_REG(RV_DMA_PERF_CTRL_OFF) = 0x3;

    // Start transfer
    RV_DMA_CH_REG(DMA_CHANNEL, RV_DMA_CH_CTRL_OFF) =
        RV_DMA_CTRL_EN | RV_DMA_CTRL_MODE_1D |
        RV_DMA_CTRL_SRC_INC | RV_DMA_CTRL_DST_INC | RV_DMA_CTRL_START;

    // Wait for completion
    while (RV_DMA_CH_REG(DMA_CHANNEL, RV_DMA_CH_STAT_OFF) & RV_DMA_STAT_BUSY);

    // Check for errors
    uint32_t stat = RV_DMA_CH_REG(DMA_CHANNEL, RV_DMA_CH_STAT_OFF);
    if (stat & RV_DMA_STAT_ERR) {
        rv_magic_exit(1);
    }

    // Clear done flag
    RV_DMA_CH_REG(DMA_CHANNEL, RV_DMA_CH_STAT_OFF) = RV_DMA_STAT_DONE;
}

int main(void) {
    // Initialize source buffer
    for (int i = 0; i < DATA_SIZE; i++) {
        src_buffer[i] = i & 0xFF;
    }

    // Perform DMA copy
    dma_memcpy(dst_buffer, src_buffer, DATA_SIZE);

    // Read performance counters
    uint32_t cycles = RV_DMA_GLB_REG(RV_DMA_PERF_CYCLES_OFF);
    uint32_t rd_bytes = RV_DMA_GLB_REG(RV_DMA_PERF_RD_BYTES_OFF);
    uint32_t wr_bytes = RV_DMA_GLB_REG(RV_DMA_PERF_WR_BYTES_OFF);

    // Verify data
    for (int i = 0; i < DATA_SIZE; i++) {
        if (dst_buffer[i] != src_buffer[i]) {
            rv_magic_exit(2);
        }
    }

    rv_magic_exit(0);
    return 0;
}

== Appendix C: Performance Considerations

=== DMA vs CPU Copy

For bulk data transfers > 256 bytes, DMA provides significant performance benefits:

* *CPU Copy*: ~4-8 cycles per word (load + store + loop overhead)
* *DMA*: 1-2 cycles per word (overlapped read/write, no CPU involvement)

=== Cache Coherency

The RV32 core includes an instruction cache but no data cache. When using DMA:

* No explicit cache maintenance required for data transfers
* Use `fence` instruction after MMIO writes to ensure ordering

=== Interrupt Latency

Typical interrupt latency (cycles from assertion to handler entry):

* Minimum: ~15 cycles (no contention)
* Average: ~25 cycles (with PLIC arbitration)
* Maximum: ~100 cycles (worst-case pipeline state)
