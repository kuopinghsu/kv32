This document provides a comprehensive reference for the KV32 SoC Software Development Kit (SDK).
It describes the API functions, register definitions, and programming model for all peripherals
and system components in the KV32 RISC-V System-on-Chip.

The SDK provides a header-only C library optimized for bare-metal embedded applications,
with zero overhead when functions are not used.

== Introduction

=== Document Scope

This API reference covers:

* Memory-mapped I/O (MMIO) peripheral drivers
* Interrupt controller configuration (PLIC, CLINT)
* Control and Status Register (CSR) access
* System initialization and exception handling
* Debug and simulation utilities

=== SDK Architecture

The KV32 SDK follows these design principles:

Header-Only Design:: All driver functions are implemented as `static inline` functions,
eliminating the need to link against a separate library. Unused functions contribute zero code size.

Zero Overhead Abstraction:: Direct register access with no additional memory or CPU overhead compared to raw MMIO operations.

Type Safety:: All register addresses and bit fields are defined as compile-time constants with proper types.

Portability:: Works with both RTL simulation and software simulators (kv32sim, Spike).

=== Header Organization

[cols="2,3,5"]
|===
| Header File | Peripheral/Module | Description

| `kv_platform.h`
| Platform definitions
| Base addresses, register offsets, bit-field constants for all peripherals

| `kv_uart.h`
| UART
| Serial communication driver (polling and interrupt modes)

| `kv_spi.h`
| SPI
| SPI master controller driver

| `kv_i2c.h`
| I2C
| I2C master controller driver

| `kv_plic.h`
| PLIC
| Platform-Level Interrupt Controller configuration

| `kv_clint.h`
| CLINT
| Core-Local Interrupt Controller (timer and software interrupts)

| `kv_irq.h`
| Interrupt Management
| Global and per-source interrupt control, handler registration

| `csr.h`
| CSR Access
| Control and Status Register read/write functions

| `kv_dma.h`
| DMA Controller
| DMA controller driver (channel management, 1D/2D/3D/SG transfers, IRQ, performance)

| `kv_timer.h`
| Timer/PWM
| Timer/PWM driver (polling and interrupt modes)

| `kv_gpio.h`
| GPIO
| General-Purpose I/O driver

| `kv_wdt.h`
| Watchdog Timer
| Hardware watchdog timer driver (start, kick, stop, IRQ clear)

| `kv_cap.h`
| Capability Constants
| Expected capability register values for all peripherals (derived from RTL parameters)
|===

=== Memory-Mapped Register Access

All peripherals use 32-bit aligned registers. The SDK provides the `KV_REG32` macro for safe MMIO access:

[source,c]
----
#define KV_REG32(base, off) \
    (*(volatile uint32_t *)((uintptr_t)(base) + (uintptr_t)(off)))
----

Example usage:

[source,c]
----
uint32_t status = KV_REG32(KV_UART_BASE, KV_UART_STATUS_OFF);
KV_REG32(KV_UART_BASE, KV_UART_DATA_OFF) = 0x41; // Write 'A'
----

== Capability Register Definitions (`kv_cap.h`)

=== Overview

The `kv_cap.h` header provides compile-time constants that mirror the RTL `CAPABILITY_REG`
localparams for each peripheral. These values can be compared at runtime against the
hardware capability register to verify RTL and SDK are in sync.

Include this header in any file that reads capability registers:

[source,c]
----
#include "kv_cap.h"
----

=== Capability Register Layout

[cols="2,3,6"]
|===
| Peripheral | Bit Layout | Expected Value

| UART
| `[31:16]` VERSION, `[15:8]` RX_FIFO_DEPTH, `[7:0]` TX_FIFO_DEPTH
| `0x00011010` (v0.0001, FIFO=16)

| I2C
| `[31:16]` VERSION, `[15:8]` RX_FIFO_DEPTH, `[7:0]` TX_FIFO_DEPTH
| `0x00010808` (v0.0001, FIFO=8)

| SPI
| `[31:16]` VERSION, `[23:20]` NUM_CS, `[19:16]` RX_FIFO(4-bit), `[7:0]` TX_FIFO_DEPTH
| `0x00014808` (v0.0001, 4 CS, FIFO=8)

| DMA
| `[31:16]` VERSION, `[15:8]` NUM_CHANNELS, `[7:0]` MAX_BURST_LEN
| `0x00010410` (v0.0001, 4 ch, burst=16)

| GPIO
| `[31:16]` VERSION, `[15:8]` NUM_BANKS, `[7:0]` NUM_PINS
| `0x00010104` (v0.0001, 1 bank, 4 pins)

| Timer
| `[31:16]` VERSION, `[15:8]` NUM_CHANNELS, `[7:0]` COUNTER_WIDTH
| `0x00010420` (v0.0001, 4 ch, 32-bit)
|===

=== Constants

[source,c]
----

== UART Driver API

=== Overview

The UART peripheral (`axi_uart.sv`) provides full-duplex serial communication with:

* Configurable baud rate
* 8-bit data, no parity, 1 stop bit (8N1 format)
* TX and RX FIFOs (16 entries each)
* Interrupt generation on RX data available and TX FIFO empty
* Internal loopback mode for testing

Base Address:: `0x2001_0000` (KV_UART_BASE)
PLIC Source ID:: 1 (KV_PLIC_SRC_UART)

=== Register Map

[cols="2,2,2,6"]
|===
| Offset | Name | Access | Description

| `0x00`
| DATA
| R/W
| TX (write) / RX (read) data register

| `0x04`
| STATUS
| RO
| Status flags (TX busy, RX ready, FIFO full)

| `0x08`
| IE
| R/W
| Interrupt Enable register

| `0x0C`
| IS
| R/W1C
| Interrupt Status register (write-1-to-clear)

| `0x10`
| LEVEL
| R/W
| Baud rate divisor

| `0x14`
| CTRL
| R/W
| Control register (loopback enable)
|===

=== Status Bits

[cols="1,2,5"]
|===
| Bit | Name | Description

| 0
| TX_BUSY / TX_FULL
| TX FIFO full – cannot write

| 2
| RX_READY
| RX FIFO not empty – data available

| 3
| RX_FULL
| RX FIFO full
|===

=== API Functions

==== Initialization

[source,c]
----
void kv_uart_init(uint32_t baud_div);
----

Initialize the UART peripheral with the specified baud rate divisor.

*Parameters:*

* `baud_div`: Baud rate divisor = `(sys_clk_hz / (baud_rate * 8)) - 1`

*Example:*

[source,c]
----
// Configure for 115200 baud at 100 MHz system clock
// baud_div = (100000000 / (115200 * 8)) - 1 = 107
kv_uart_init(107);
----

==== Status Queries

[source,c]
----
int kv_uart_tx_busy(void);    // Returns 1 if TX FIFO full
int kv_uart_rx_ready(void);   // Returns 1 if RX data available
----

==== Polling Transmit

[source,c]
----
void kv_uart_putc(char c);
void kv_uart_puts(const char *s);
void kv_uart_write(const uint8_t *buf, uint32_t len);
----

*Description:*

* `kv_uart_putc()`: Block until TX FIFO has space, then transmit one character
* `kv_uart_puts()`: Transmit a null-terminated string
* `kv_uart_write()`: Transmit len bytes from buffer

*Example:*

[source,c]
----
kv_uart_puts("Hello, World!\n");
----

==== Polling Receive

[source,c]
----
int kv_uart_getc(void);              // Non-blocking, returns -1 if empty
uint8_t kv_uart_getc_blocking(void); // Blocks until data available
----

*Example:*

[source,c]
----
int c;
while ((c = kv_uart_getc()) != -1) {
    kv_uart_putc((char)c);  // Echo received character
}
----

==== Interrupt Control

[source,c]
----
void kv_uart_irq_enable(uint32_t mask);
void kv_uart_irq_disable(uint32_t mask);
uint32_t kv_uart_irq_status(void);
----

*Interrupt Sources:*

* `KV_UART_IE_RX_READY` (bit 0): RX FIFO not empty
* `KV_UART_IE_TX_EMPTY` (bit 1): TX FIFO drained (empty)

*Example:*

[source,c]
----
// Enable RX interrupt
kv_uart_irq_enable(KV_UART_IE_RX_READY);

// In interrupt handler:
uint32_t status = kv_uart_irq_status();
if (status & KV_UART_IE_RX_READY) {
    char c = kv_uart_getc();
    // Process received character
}
----

==== Loopback Mode

[source,c]
----
void kv_uart_loopback_enable(void);
void kv_uart_loopback_disable(void);
----

Enable internal TX→RX loopback for hardware testing. Transmitted bytes are fed back to the RX path without leaving the chip.

==== Capability Register

[source,c]
----
uint32_t kv_uart_get_capability(void);    // Full 32-bit capability register
uint32_t kv_uart_get_tx_fifo_depth(void); // TX FIFO depth in bytes
uint32_t kv_uart_get_rx_fifo_depth(void); // RX FIFO depth in bytes
uint32_t kv_uart_get_version(void);       // Hardware version [31:16]
----

Reads the read-only capability register at offset `KV_UART_CAP_OFF` (`0x18`).
Compare against `KV_CAP_UART_VALUE` from `kv_cap.h` to verify hardware matches SDK.

[source,c]
----
#include "kv_cap.h"
uint32_t cap = kv_uart_get_capability();
printf("UART cap: 0x%08X (expected 0x%08X)\n", cap, KV_CAP_UART_VALUE);
printf("TX FIFO: %u  RX FIFO: %u\n",
       kv_uart_get_tx_fifo_depth(), kv_uart_get_rx_fifo_depth());
----

'''

== SPI Driver API

=== Overview

The SPI peripheral (`axi_spi.sv`) provides a master-mode SPI controller with:

* Configurable clock frequency
* All four SPI modes (CPOL, CPHA)
* Up to 4 independent chip-select lines
* TX and RX FIFOs (16 entries each)
* Full-duplex operation
* Internal loopback mode

Base Address:: `0x2003_0000` (KV_SPI_BASE)
PLIC Source ID:: 2 (KV_PLIC_SRC_SPI)

=== Register Map

[cols="2,2,2,6"]
|===
| Offset | Name | Access | Description

| `0x00`
| CTRL
| R/W
| Control register (enable, CPOL, CPHA, CS, loopback)

| `0x04`
| DIV
| R/W
| Clock divisor

| `0x08`
| TX
| WO
| TX FIFO push

| `0x0C`
| RX
| RO
| RX FIFO pop

| `0x10`
| STATUS
| RO
| Status flags

| `0x14`
| IE
| R/W
| Interrupt Enable

| `0x18`
| IS
| R/W1C
| Interrupt Status (same bit map as IE; level-driven)
|===

=== Register Bit Fields

==== CTRL Register (`0x00`)

[cols="1,2,5"]
|===
| Bit(s) | Name | Description

| 0
| ENABLE
| Enable SPI controller

| 1
| CPOL
| Clock idle polarity

| 2
| CPHA
| Clock phase

| 3
| LOOPBACK
| Internal MOSI->MISO loopback enable

| 7:4
| CS[3:0]
| Chip-select outputs (active-low in hardware; 1 = deasserted, 0 = asserted)

| 31:8
| Reserved
| Read as 0, ignore on write
|===

==== STATUS Register (`0x10`)

[cols="1,2,5"]
|===
| Bit | Name | Description

| 0 | BUSY     | Transfer in progress
| 1 | TX_READY | TX FIFO can accept data (not full)
| 2 | RX_VALID | RX FIFO has data
| 3 | TX_EMPTY | TX FIFO empty
| 4 | RX_FULL  | RX FIFO full
| 31:5 | Reserved | Read as 0
|===

==== IE / IS Registers (`0x14` / `0x18`)

[cols="1,2,5"]
|===
| Bit | Name | Description

| 0 | RX_READY | RX FIFO not empty interrupt/status
| 1 | TX_EMPTY | TX FIFO empty interrupt/status
| 31:2 | Reserved | Read as 0
|===

==== TX / RX Data Registers (`0x08` / `0x0C`)

[cols="1,2,5"]
|===
| Bit(s) | Name | Description

| 7:0
| DATA
| Transmit byte (TX) / received byte (RX)

| 31:8
| Reserved
| Read as 0, ignore on write
|===

=== SPI Modes

[cols="1,1,1,4"]
|===
| Mode | CPOL | CPHA | Description

| MODE0
| 0
| 0
| Clock idle low, sample on rising edge

| MODE1
| 0
| 1
| Clock idle low, sample on falling edge

| MODE2
| 1
| 0
| Clock idle high, sample on falling edge

| MODE3
| 1
| 1
| Clock idle high, sample on rising edge
|===

=== API Functions

==== Initialization

[source,c]
----
void kv_spi_init(uint32_t clk_div, uint32_t mode);
----

*Parameters:*

* `clk_div`: SCK frequency = `sys_clk / (2 * (clk_div + 1))`
* `mode`: `KV_SPI_MODE0`, `MODE1`, `MODE2`, or `MODE3`

*Example:*

[source,c]
----
// Configure for 1 MHz SPI clock at 100 MHz system clock, mode 0
// clk_div = (100000000 / (2 * 1000000)) - 1 = 49
kv_spi_init(49, KV_SPI_MODE0);
----

==== Chip Select Control

[source,c]
----
void kv_spi_cs_select(uint32_t n);   // Assert CS[n] (0-3)
void kv_spi_cs_deselect(void);       // Deassert all CS lines
----

*Example:*

[source,c]
----
kv_spi_cs_select(0);        // Assert CS0
// Perform transfer
kv_spi_cs_deselect();       // Release CS0
----

==== Data Transfer

[source,c]
----
uint8_t kv_spi_transfer(uint8_t data);
void kv_spi_transfer_buf(const uint8_t *tx_buf, uint8_t *rx_buf, uint32_t len);
void kv_spi_read_buf(uint8_t *rx_buf, uint32_t len);
----

*Description:*

* `kv_spi_transfer()`: Full-duplex transfer of one byte
* `kv_spi_transfer_buf()`: Transfer multiple bytes (tx_buf or rx_buf may be NULL)
* `kv_spi_read_buf()`: Read len bytes by clocking out 0xFF

*Example:*

[source,c]
----
uint8_t tx_data[] = {0x03, 0x00, 0x00, 0x00}; // Read command + address
uint8_t rx_data[256];

kv_spi_cs_select(0);
kv_spi_transfer_buf(tx_data, NULL, 4);  // Send command
kv_spi_read_buf(rx_data, 256);          // Read data
kv_spi_cs_deselect();
----

==== Status and Control

[source,c]
----
int kv_spi_busy(void);        // Transfer in progress
int kv_spi_tx_ready(void);    // Can accept TX data
int kv_spi_rx_valid(void);    // RX data available
void kv_spi_wait_ready(void); // Block until idle
----

==== Interrupt Control

[source,c]
----
void kv_spi_irq_enable(uint32_t mask);
void kv_spi_irq_disable(uint32_t mask);
uint32_t kv_spi_irq_status(void);
----

*Interrupt Sources:*

* `KV_SPI_IE_RX_READY` (bit 0): RX FIFO not empty
* `KV_SPI_IE_TX_EMPTY` (bit 1): TX FIFO empty

==== Loopback Mode

[source,c]
----
void kv_spi_loopback_enable(void);
void kv_spi_loopback_disable(void);
----

==== Capability Register

[source,c]
----
uint32_t kv_spi_get_capability(void);    // Full 32-bit capability register
uint32_t kv_spi_get_tx_fifo_depth(void); // TX FIFO depth
uint32_t kv_spi_get_rx_fifo_depth(void); // RX FIFO depth
uint32_t kv_spi_get_num_cs(void);        // Number of chip-select lines
uint32_t kv_spi_get_version(void);       // Hardware version [31:16]
----

Reads the read-only capability register at offset `KV_SPI_CAP_OFF` (`0x1C`).
Compare against `KV_CAP_SPI_VALUE` from `kv_cap.h`.

[source,c]
----
#include "kv_cap.h"
uint32_t cap = kv_spi_get_capability();
printf("SPI cap: 0x%08X  CS=%u TX_FIFO=%u\n",
       cap, kv_spi_get_num_cs(), kv_spi_get_tx_fifo_depth());
----

== I2C Driver API

=== Overview

The I2C peripheral (`axi_i2c.sv`) provides a master-mode I2C controller with:

* Configurable clock frequency (100 kHz / 400 kHz / custom)
* 7-bit addressing
* START/STOP condition generation
* ACK/NACK detection
* Interrupt support

Base Address:: `0x2002_0000` (KV_I2C_BASE)
PLIC Source ID:: 3 (KV_PLIC_SRC_I2C)

=== Register Map

[cols="2,2,2,6"]
|===
| Offset | Name | Access | Description

| `0x00`
| CTRL
| R/W
| Control register (enable, START, STOP, READ, NACK)

| `0x04`
| DIV
| R/W
| Clock divisor

| `0x08`
| TX
| WO
| Transmit data register

| `0x0C`
| RX
| RO
| Receive data register

| `0x10`
| STATUS
| RO
| Status flags (busy, ACK received, RX valid)

| `0x14`
| IE
| R/W
| Interrupt Enable

| `0x18`
| IS
| R/W1C
| Interrupt Status
|===

=== Register Bit Fields

==== CTRL Register (`0x00`)

[cols="1,2,5"]
|===
| Bit(s) | Name | Description

| 0 | ENABLE | Enable I2C controller
| 1 | START  | Issue START condition
| 2 | STOP   | Issue STOP condition
| 3 | READ   | Read byte operation (vs write)
| 4 | NACK   | Send NACK after received byte
| 31:5 | Reserved | Read as 0, ignore on write
|===

==== STATUS Register (`0x10`)

[cols="1,2,5"]
|===
| Bit | Name | Description

| 0 | BUSY     | Transfer in progress
| 1 | TX_READY | TX path can accept new byte
| 2 | RX_VALID | RX data available
| 3 | ACK_RECV | Last transmitted byte was ACKed by slave
| 31:4 | Reserved | Read as 0
|===

==== IE / IS Registers (`0x14` / `0x18`)

[cols="1,2,5"]
|===
| Bit | Name | Description

| 0 | RX_READY  | RX FIFO not empty interrupt/status
| 1 | TX_EMPTY  | TX FIFO empty interrupt/status
| 2 | STOP_DONE | STOP condition completed (`IS` bit is sticky and cleared by W1C)
| 31:3 | Reserved | Read as 0
|===

==== TX / RX Data Registers (`0x08` / `0x0C`)

[cols="1,2,5"]
|===
| Bit(s) | Name | Description

| 7:0
| DATA
| Transmit byte (TX) / received byte (RX)

| 31:8
| Reserved
| Read as 0, ignore on write
|===

=== API Functions

==== Initialization

[source,c]
----
void kv_i2c_init(uint32_t clk_div);
----

*Parameters:*

* `clk_div`: SCL period = `sys_clk / (4 * (clk_div + 1))`

*Example:*

[source,c]
----
// Configure for 100 kHz at 100 MHz system clock
// clk_div = (100000000 / (4 * 100000)) - 1 = 249
kv_i2c_init(249);

// Configure for 400 kHz at 100 MHz system clock
// clk_div = (100000000 / (4 * 400000)) - 1 = 62
kv_i2c_init(62);
----

==== Low-Level Operations

[source,c]
----
void kv_i2c_start(void);                    // Issue START condition
void kv_i2c_stop(void);                     // Issue STOP condition
int kv_i2c_write_byte(uint8_t data);        // Write byte, return 0 if ACK
uint8_t kv_i2c_read_byte(int send_ack);     // Read byte, send ACK if 1
----

==== High-Level Transfers

[source,c]
----
int kv_i2c_master_write(uint8_t addr, const uint8_t *buf, uint32_t len);
int kv_i2c_master_read(uint8_t addr, uint8_t *buf, uint32_t len);
----

*Description:*

Complete write or read transactions including START, address, data, and STOP.

*Parameters:*

* `addr`: 7-bit I2C device address
* `buf`: Data buffer
* `len`: Number of bytes to transfer

*Return Values:*

* `0`: Success
* `-1`: Device did not ACK the address
* `-2`: Device did not ACK a data byte

*Example:*

[source,c]
----
// Write to EEPROM at address 0x50
uint8_t eeprom_addr = 0x50;
uint8_t write_data[] = {0x00, 0x10, 0xAA, 0xBB}; // Addr MSB, LSB, data

if (kv_i2c_master_write(eeprom_addr, write_data, 4) < 0) {
    // Handle error
}

// Read from EEPROM
uint8_t read_buf[2];
if (kv_i2c_master_read(eeprom_addr, read_buf, 2) < 0) {
    // Handle error
}
----

==== Status and Control

[source,c]
----
int kv_i2c_busy(void);           // Transfer in progress
int kv_i2c_ack_received(void);   // Slave ACKed last byte
int kv_i2c_rx_valid(void);       // RX data available
void kv_i2c_wait_ready(void);    // Block until idle
----

==== Interrupt Control

[source,c]
----
void kv_i2c_irq_enable(uint32_t mask);
void kv_i2c_irq_disable(uint32_t mask);
uint32_t kv_i2c_irq_status(void);
----

*Interrupt Sources:*

* `KV_I2C_IE_RX_READY` (bit 0): RX FIFO not empty
* `KV_I2C_IE_TX_EMPTY` (bit 1): TX FIFO empty
* `KV_I2C_IE_STOP_DONE` (bit 2): STOP condition completed (sticky until W1C clear)

`kv_i2c_irq_status()` reads `IS` and writes the same value back.
This acknowledges `IS[2]` (W1C STOP_DONE); `IS[1:0]` are level-driven and
clear when the underlying FIFO/busy condition deasserts.

==== Capability Register

[source,c]
----
uint32_t kv_i2c_get_capability(void);    // Full 32-bit capability register
uint32_t kv_i2c_get_tx_fifo_depth(void); // TX FIFO depth
uint32_t kv_i2c_get_rx_fifo_depth(void); // RX FIFO depth
uint32_t kv_i2c_get_version(void);       // Hardware version [31:16]
----

Reads the read-only capability register at offset `KV_I2C_CAP_OFF` (`0x1C`).

== DMA Controller API

=== Overview

The DMA controller (`axi_dma.sv`) provides high-performance data transfers with:

* 4 independent channels
* Multiple transfer modes: 1D, 2D strided, 3D planar, scatter-gather

* Interrupt generation on completion or error
* Performance counters for throughput monitoring

=== Transfer Modes

==== 1D Transfer Mode

Simple linear memory copy using `kv_dma_1d_copy()`:

[source,c]
----
#include "kv_dma.h"

// Copy 1024 bytes from src to dst on channel 0, then verify done
int ret = kv_dma_1d_copy(0, src_addr, dst_addr, 1024);
if (ret != 0) { /* AXI error */ }
kv_dma_ch_reset(0);  // Clear DONE/ERR flags
----

For non-blocking use, set up the channel manually and call `kv_dma_ch_start()`:

[source,c]
----
kv_dma_ch_reset(0);
KV_DMA_CH_REG(0, KV_DMA_CH_SRC_OFF)  = src_addr;
KV_DMA_CH_REG(0, KV_DMA_CH_DST_OFF)  = dst_addr;
KV_DMA_CH_REG(0, KV_DMA_CH_XFER_OFF) = 1024;
KV_DMA_CH_REG(0, KV_DMA_CH_CTRL_OFF) = KV_DMA_CTRL_EN | KV_DMA_CTRL_MODE_1D
                                       | KV_DMA_CTRL_SRC_INC | KV_DMA_CTRL_DST_INC;
kv_dma_ch_start(0);
// ... do other work ...
kv_dma_ch_wait(0);   // Blocks until done or error
kv_dma_ch_reset(0);
----

==== 2D Strided Transfer Mode

Copy rectangular regions with different pitches using `kv_dma_2d_copy()`:

[source,c]
----
// Copy 640x480 image; src stride=1024 bytes/row, dst stride=800 bytes/row

    src_addr, dst_addr,
    640,   /* row_bytes  */
    480,   /* num_rows   */
    1024,  /* src_stride */
    800);  /* dst_stride */

----

==== 3D Planar Transfer Mode

Copy 3D data structures (e.g., video frames) using `kv_dma_3d_copy()`:

[source,c]
----
// Copy video buffer: 640x480x3 (RGB planes)
// Each plane: 640 bytes/row, 480 rows; planes packed contiguously
int ret = kv_dma_3d_copy(0,
    src_addr, dst_addr,
    640,    /* row_bytes    */
    480,    /* num_rows     */
    3,      /* num_planes   */
    640,    /* src_stride   */
    640,    /* dst_stride   */

==== Scatter-Gather Transfer Mode

Process a descriptor list using `kv_dma_sg_copy()`:

[source,c]

== Timer Driver API

=== Overview

The Timer (`axi_timer.sv`) provides 4 independent 32-bit up-counters with PWM output,
prescaler, dual compare, and per-channel interrupt support.

Base Address:: `0x2004_0000` (KV_TIMER_BASE)
PLIC Sources:: 6–9 (`KV_PLIC_SRC_TIMER0` through `KV_PLIC_SRC_TIMER3` — one per channel)
Header:: `kv_timer.h`

=== Register Map

Each timer channel occupies 0x20 bytes starting at `BASE + channel * 0x20`.

[cols="2,1,1,5"]
|===
| Register | Offset | Access | Description

| COUNT
| 0x00
| R/W
| Current counter value (resets on COMPARE2 match)

| COMPARE1
| 0x04
| R/W
| Compare 1 threshold (interrupt / PWM rising edge)

| COMPARE2
| 0x08
| R/W
| Compare 2 threshold (period end + reload)

| CTRL
| 0x0C
| R/W
| Control register (see bit table below)
|===

Global registers (shared across all channels):

[cols="2,1,1,5"]
|===
| Register | Offset | Access | Description

| INT_STATUS
| 0x80
| W1C
| Interrupt status — one bit per channel (bits 3:0)

| INT_ENABLE
| 0x84
| R/W
| Interrupt enable — one bit per channel (bits 3:0)

| CAPABILITY
| 0x88
| RO
| `{VERSION[31:16], NUM_CHANNELS[15:8], COUNTER_WIDTH[7:0]}`
|===

CTRL bit fields:

[cols="1,1,5"]
|===
| Bit | Name | Description

| 0 | EN | Timer enable
| 1 | PWM_EN | PWM mode enable
| 3 | INT_EN | Interrupt enable
| 4 | PWM_POL | PWM polarity (1 = active high)
| [31:16] | PRESCALE | Clock prescaler (counts at clock / (PRESCALE + 1))
|===

== GPIO Driver API

=== Overview

The GPIO controller (`axi_gpio.sv`) provides 4 independent 32-pin banks with
configurable direction, atomic set/clear, edge/level interrupt detection,

PLIC Source:: 5 (KV_PLIC_SRC_GPIO)
Header:: `kv_gpio.h`

=== Register Map

Register banks are organized per pin-bank (banks 0–3, 32 pins each).

[cols="2,1,1,5"]
|===
| Register | Bank-0 Offset | Access | Description

| DATA_OUT | 0x00–0x0C | R/W | Output data (bank 0–3 at +0,+4,+8,+C)
| SET      | 0x10–0x1C | WO  | Atomic set bits (write-1-to-set)
| CLEAR    | 0x20–0x2C | WO  | Atomic clear bits (write-1-to-clear)
| DATA_IN  | 0x30–0x3C | RO  | Input data (read current pin state)
| DIR      | 0x40–0x4C | R/W | Direction (1=output, 0=input)

| TRIGGER  | 0x60–0x6C | R/W | Trigger mode (1=edge, 0=level)
| POLARITY | 0x70–0x7C | R/W | Polarity (edge: 1=rising; level: 1=high)
| IS       | 0x80–0x8C | W1C | Interrupt status
| LOOPBACK | 0x90–0x9C | R/W | Loopback enable (output → input path)
| CAPABILITY | 0xA0    | RO  | `{VERSION[31:16], NUM_BANKS[15:8], NUM_PINS[7:0]}`
|===

=== Register Bit-Field Conventions

[cols="2,1,5"]

| DATA_OUT
| 31:0

| 31:0

| CLEAR
| 31:0
| Write-1-to-clear output bits atomically

| DATA_IN
| 31:0
| Live sampled input pin state

| DIR
| 31:0
| Direction control (`1` = output, `0` = input)

| IE
| 31:0
| Interrupt enable mask (`1` enables interrupt on corresponding pin)

| TRIGGER
| 31:0
| Trigger type (`1` = edge-sensitive, `0` = level-sensitive)

| POLARITY
| 31:0

| IS
| 31:0
| Interrupt status (write-1-to-clear)

| LOOPBACK
| 31:0

[cols="1,2,5"]

== Watchdog Timer (WDT) Driver API

=== Overview

The WDT (`axi_wdt.sv`) counts down from a software-configured LOAD value and, on expiry, either raises a PLIC interrupt (IRQ mode) or asserts the `wdt_reset_o` hardware reset output (reset mode).

Base Address:: `0x2006_0000` (KV_WDT_BASE)
PLIC Source:: 10 (`KV_PLIC_SRC_WDT`) — IRQ mode only
Header:: `kv_wdt.h`

=== Register Map

[cols="2,2,2,6"]
|===
| Offset | Name | Access | Description

| `0x00`
| CTRL
| R/W
| Control (`EN`, `INTR_EN`)

| `0x04`
| LOAD
| R/W
| Reload value written by software

| `0x08`
| COUNT
| RO
| Current countdown value

| `0x0C`
| KICK
| WO
| Reload COUNT from LOAD (written value ignored)

| `0x10`
| STATUS
| R/W1C
| Status (`WDT_INT`)

| `0x14`
| CAP
| RO
| Capability constant `0x0001_0020` (version 0.1, 32-bit counter)
|===

=== Register Bit Fields

==== CTRL Register (`0x00`)

[cols="1,2,5"]
|===
| Bit(s) | Name | Description

| 0 | EN      | Watchdog enable
| 1 | INTR_EN | Expiry action: `1` = interrupt, `0` = hardware reset request
| 31:2 | Reserved | Read as 0, ignore on write
|===

==== STATUS Register (`0x10`)

[cols="1,2,5"]
|===
| Bit(s) | Name | Description

| 0
| WDT_INT
| Watchdog expiry interrupt/status (write-1-to-clear)

| 31:1
| Reserved
| Read as 0
|===

==== LOAD / COUNT Registers (`0x04` / `0x08`)

[cols="1,2,5"]
|===
| Bit(s) | Name | Description

| 31:0
| VALUE
| Countdown reload (LOAD) / live counter value (COUNT)
|===

=== Quick Reference

[source,c]
----
// Start WDT: set reload value and expiry action.
//   load    - countdown start value (cycles)
//   intr_en - 1 = IRQ on expiry (PLIC src 10); 0 = hardware reset
void kv_wdt_start(uint32_t load, int intr_en);

// Reload COUNT from LOAD ("pet" the watchdog).
void kv_wdt_kick(void);

// Disable the WDT (clear EN bit).
void kv_wdt_stop(void);

// Clear WDT_INT status bit (W1C).  Call from ISR before kv_plic_complete().
void kv_wdt_clear_int(void);
----

=== Usage — IRQ Mode

[source,c]
----
#include "kv_wdt.h"
#include "kv_plic.h"
#include "kv_irq.h"

static void wdt_handler(uint32_t cause) {
    (void)cause;
    uint32_t src = kv_plic_claim();
    if (src == KV_PLIC_SRC_WDT) {
        kv_wdt_clear_int();
        kv_wdt_kick();
    }
    kv_plic_complete(src);
}

void setup_wdt_irq(void) {
    kv_irq_register(KV_CAUSE_MEI, wdt_handler);
    kv_plic_init_source(KV_PLIC_SRC_WDT, 1);
    kv_irq_enable();
    kv_wdt_start(100000u, 1);   /* LOAD=100000, INTR_EN=1 */
}
----

=== Usage — Reset Mode

[source,c]
----

== Exception Handling and Trap Frame

=== Overview

When a machine-mode trap fires the processor saves the machine state and jumps to
`mtvec`, which points to the assembly stub `trap_vector` in `start.S`. That stub:

1. Allocates a 144-byte `kv_trap_frame_t` on the interrupted thread's stack.
2. Saves all 32 general-purpose registers and four CSRs (`mepc`, `mstatus`, `mcause`, `mtval`) into the frame.
3. Passes a **pointer to the frame** as the sole argument to `trap_handler()`.
4. On return, restores machine state from the frame, then executes `mret`.

Because the epilogue restores `mepc` from `frame->mepc`, exception handlers redirect
the return PC by writing `frame->mepc` — no direct `csrw mepc` inside the handler.

=== Trap Frame Layout (`kv_trap_frame_t`)

Defined in `kv_irq.h`. Total size: 144 bytes, allocated on the interrupted thread's
stack. The frame is the *authoritative* state used by `mret`.

[cols="1,2,3,6"]
|===
| Offset | Field | Register | Notes

| +0   | `_pad`    | x0          | Placeholder; x0 is hardwired zero
| +4   | `ra`      | x1          | Return address at trap entry
| +8   | `sp`      | x2          | Stack pointer at trap entry (post-allocation)
| +12  | `gp`      | x3          | Global pointer
| +16  | `tp`      | x4          | Thread pointer
| +20  | `t0`      | x5          | Temporary
| +24  | `t1`      | x6          | Temporary
| +28  | `t2`      | x7          | Temporary
| +32  | `s0`      | x8 / fp     | Saved register / frame pointer
| +36  | `s1`      | x9          | Saved register
| +40  | `a0`      | x10         | Argument / return value
| +44  | `a1`      | x11         | Argument / return value
| +48  | `a2`      | x12         | Argument
| +52  | `a3`      | x13         | Argument
| +56  | `a4`      | x14         | Argument
| +60  | `a5`      | x15         | Argument
| +64  | `a6`      | x16         | Argument
| +68  | `a7`      | x17         | Argument / syscall number (ecall convention)
| +72  | `s2`–`s11`| x18–x27     | Saved registers (at offsets +72…+108)
| +112 | `t3`–`t6` | x28–x31     | Temporaries (at offsets +112…+124)
| +128 | `mepc`    | mepc CSR    | *Writable* — write to redirect the `mret` return PC
| +132 | `mstatus` | mstatus CSR | Restored verbatim; read-only for normal handlers
| +136 | `mcause`  | mcause CSR  | Read-only; MSB set for interrupts, clear for exceptions
| +140 | `mtval`   | mtval CSR   | Read-only; faulting address (load/store) or bad instruction
|===

TIP: The field names match the RISC-V ABI register names (`ra`, `sp`, `a0`…`a7`, `s0`…`s11`,
`t0`…`t6`), so accessing a saved argument register in a handler is simply `frame->a0`.

=== Design Rationale — Frame Pointer Instead of Direct CSR Writes

==== Why the Old Approach Was Fragile

The previous interface passed `(mcause, mepc, mtval)` as plain scalars and expected
exception handlers to call `write_csr_mepc(mepc + 4)` to skip a faulting instruction.
That relied on the trap epilogue re-reading `mepc` from the live CSR just before `mret`.
Two conditions break that assumption:

Nested interrupts:: A higher-priority interrupt can preempt an executing handler and
overwrite the `mepc` CSR with the preempted handler's PC. When the outer handler later
calls `write_csr_mepc()`, its value is correct — but if the epilogue reads `mepc` from
the frame (required by the nested-interrupt fix), that CSR write is ignored.

Epilogue restore ordering:: The epilogue saves `mepc` in the frame and restores it
from there to support nesting correctly. Any `write_csr_mepc()` call inside the handler
is silently overwritten by `lw t0, 128(sp); csrw mepc, t0` on the way out.
This was the root cause of the bus_err infinite-loop bug: the handler updated the CSR
to skip the faulting instruction, but the epilogue restored the original (faulting)
`mepc`, causing the same instruction to re-execute forever.

==== The Fix: Frame Is the Authoritative Source

The epilogue has one invariant: it reads `mepc` from `frame->mepc` and writes it to the
CSR. Exception handlers that want to redirect execution write `frame->mepc`:

[source,c]
----
// WRONG — CSR write is overwritten by the epilogue's frame→CSR restore
static void bad_handler(kv_trap_frame_t *frame) {
    write_csr_mepc(frame->mepc + 4);   // ← clobbered on exit
}

// CORRECT — update the frame; epilogue copies frame->mepc to CSR before mret
static void good_handler(kv_trap_frame_t *frame) {
    frame->mepc += 4;                  // ← survives the epilogue
}
----

==== Additional Benefits

Single source of truth:: What is in `frame->mepc` at handler return is exactly what
`mret` uses. No reasoning about CSR side-effects or epilogue ordering required.

Debuggability:: A panic handler or debugger sees the full machine state (all 32 GPRs
plus four CSRs) behind one pointer. Stack traces can walk `frame->ra` and `frame->sp`
without probing live hardware CSRs.

Register injection:: An advanced handler (e.g. an `ecall` emulation layer) can write
arbitrary values into `frame->a0`…`frame->a7` to synthesise return values, or rewrite
the entire register context to emulate an instruction.

Nested-interrupt safety:: Re-enabling interrupts inside a handler is safe regardless of
nesting depth: each level has its own frame on the stack. Inner trap activity cannot
corrupt the `mepc`/`mstatus` saved in outer frames.

==== Interaction with the RTOS Port

The mini-RTOS (`mrtos_riscv.c`) has its own `mrtos_trap_vector` with a 132-byte
context frame optimised for fast task switching. Its `mrtos_trap_handler()` constructs
a temporary `kv_trap_frame_t` on the C stack when forwarding to `kv_irq_dispatch()`,
and calls `write_csr_mepc(frame.mepc)` to commit any exception-handler update before
the RTOS epilogue restores from its own frame. This bridges the two layouts without
merging them.

=== Writing Exception Handlers

Register with `kv_exc_register(cause, handler)`. The handler receives
`kv_trap_frame_t *frame` and returns `void`. The epilogue restores machine state from the
frame when the handler returns.

==== Skip a Faulting Instruction

[source,c]
----
static void bus_err_handler(kv_trap_frame_t *frame) {
    g_bus_err_caught = 1;
    g_bus_err_mcause = frame->mcause;  // KV_EXC_LOAD_FAULT or KV_EXC_STORE_FAULT

    /* Bits [1:0] != 0b11  →  16-bit RVC instruction, else 32-bit */
    uint16_t inst16 = *(volatile uint16_t *)frame->mepc;
    frame->mepc += ((inst16 & 0x3u) != 0x3u) ? 2u : 4u;
}

kv_exc_register(KV_EXC_LOAD_FAULT,  bus_err_handler);
kv_exc_register(KV_EXC_STORE_FAULT, bus_err_handler);
----

==== Redirect to a Pre-Stored Resume Address

When `mepc` itself is invalid (e.g. a null-function-call fault where `mepc = 0`),
`mepc + N` is still unmapped. Store a safe resume address before the dangerous call:

[source,c]
----
static volatile uintptr_t g_resume;

static void insn_fault_handler(kv_trap_frame_t *frame) {
    g_caught = 1;
    frame->mepc = (uint32_t)g_resume;  // redirect to pre-stored address
}

kv_exc_register(KV_EXC_INSN_FAULT, insn_fault_handler);

// Capture the instruction immediately following the dangerous call:
uintptr_t resume_pc;
asm volatile ("la %0, 1f" : "=r"(resume_pc));
g_resume = resume_pc;
((void (*)(void))NULL)();   // call NULL → KV_EXC_INSN_FAULT, mepc = 0
asm volatile ("1:");         // handler redirects mret here
----

==== Emulate an ecall (Syscall ABI)

[source,c]
----
static void ecall_handler(kv_trap_frame_t *frame) {
    uint32_t nr = frame->a7;                 // syscall number in a7 (RISC-V convention)
    frame->a0   = dispatch_syscall(nr,        // return value in a0
                                   frame->a0, frame->a1, frame->a2);
    frame->mepc += 4;                         // ecall is always 4 bytes
}

kv_exc_register(KV_EXC_ECALL_M, ecall_handler);
----

=== Writing Interrupt Handlers

=== Overriding `trap_handler` Entirely

`trap_handler()` is a `weak` symbol. Provide your own definition to replace the
`kv_irq_register` / `kv_exc_register` dispatch table:

[source,c]
----
// my_test.c — replaces the default trap_handler for the linked binary
void trap_handler(kv_trap_frame_t *frame) {
    if (frame->mcause & 0x80000000u) {
        my_fast_irq_dispatch(frame);
    } else {
        printf("EXC cause=%u pc=0x%08x val=0x%08x\n",
               frame->mcause, frame->mepc, frame->mtval);
        frame->mepc += 4;   // skip and continue
    }
}
----

NOTE: Override `trap_handler` only when the registration table is too slow or inflexible
(e.g. performance-critical bare-metal interrupt paths). For normal use,
`kv_irq_register` and `kv_exc_register` compose well and leave room to add handlers
later without rewriting existing code.

=== Trap Entry / Exit Flow

[source]
----
Trap fires (interrupt or exception)
    │
    ▼
trap_vector  (start.S, 4-byte aligned)
    │
    ├─ addi sp, sp, -144          allocate kv_trap_frame_t on stack
    ├─ sw x1..x31, 4..124(sp)     save all general-purpose registers
    ├─ csrr+sw mepc   → [sp+128]  save mepc
    ├─ csrr+sw mstatus→ [sp+132]  save mstatus
    ├─ csrr+sw mcause → [sp+136]  save mcause
    ├─ csrr+sw mtval  → [sp+140]  save mtval
    ├─ mv a0, sp                  pass frame pointer as argument
    └─ call trap_handler
               │
               ▼
           trap_handler()  [trap.c — weak symbol, override per binary]
               └─ kv_irq_dispatch(frame)
                      │
                      ├─ mcause[31]=1 → irq_table[code](code)
                      │                 interrupt handler; frame unchanged
                      └─ mcause[31]=0 → exc_table[code](frame)
                                        frame->mepc writable to redirect mret
               │
               ▼  (return from trap_handler)
    ├─ lw t0, 132(sp); csrw mstatus, t0  restore mstatus
    ├─ lw t0, 128(sp); csrw mepc, t0     restore mepc ← from frame (authoritative)
    ├─ lw x1..x31, 4..124(sp)            restore GPRs
    ├─ addi sp, sp, 144                  free frame
    └─ mret                              resume at frame->mepc
    │
    ▼
Interrupted code resumes (or redirected PC if handler wrote frame->mepc)
----

== Magic Device (Simulation/Test)

=== Overview

The Magic Device provides a simple interface for firmware to communicate with simulation environments without requiring a full UART implementation.

Base Address:: `0x4000_0000` (KV_MAGIC_BASE)

NOTE: This peripheral is only available in simulation (RTL testbench, kv32sim, Spike plugins).
It will not be synthesized for production FPGA/ASIC.

=== Registers

[cols="2,2,6"]
|===
| Offset | Name | Description

| `0x0000`
| CONSOLE
| Write character (low byte) to simulation console

| `0x0004`
| EXIT
| Write exit code to signal simulation end

| `0x1000-0x11FF`
| NCM[127:0]
| Non-cacheable memory window (`KV_NCM_BASE`, `KV_NCM_SIZE`) for uncached code/data tests
|===

=== AXI Decode and Responses

The RTL `axi_magic` slave acknowledges only these decoded regions:

* `KV_MAGIC_CONSOLE_ADDR` (`0x4000_0000`)
* `KV_MAGIC_EXIT_ADDR` (`0x4000_0004`)
* `KV_NCM_BASE .. KV_NCM_BASE + KV_NCM_SIZE - 1` (`0x4000_1000-0x4000_11FF`)

Transactions outside these regions return AXI `SLVERR`.

NCM details:

* Capacity: 128 x 32-bit words (512 bytes)
* Word index: `addr[8:2]`
* Read: returns stored 32-bit word
* Write: stores full `wdata` word (no per-byte strobe merge in RTL)

=== Exit Code Encoding

The EXIT register uses HTIF tohost encoding:

* Success (code 0): Write `1`
* Failure (code N): Write `(N << 1) \| 1`

=== API Functions

[source,c]
----
void kv_magic_putc(char c);
void kv_magic_exit(int code);
----

*Description:*

* `kv_magic_putc()`: Output one character to simulation console
* `kv_magic_exit()`: Signal program exit and halt (infinite loop)

Related constants from `kv_platform.h`:

* `KV_MAGIC_BASE`, `KV_MAGIC_CONSOLE_ADDR`, `KV_MAGIC_EXIT_ADDR`
* `KV_NCM_BASE`, `KV_NCM_SIZE`

*Example:*

[source,c]
----
#include "kv_platform.h"

int main(void) {
    const char *msg = "Hello from KV32!\n";

    // Output string
    for (const char *p = msg; *p; p++) {
        kv_magic_putc(*p);
    }

    // Signal successful completion
    kv_magic_exit(0);  // Simulation will stop here

    return 0;
}

== Complete Examples

=== Example 1: UART Echo with Interrupts

[source,c]
----
#include "kv_platform.h"
#include "kv_uart.h"
#include "kv_plic.h"
#include "kv_irq.h"

#define UART_BAUD_DIV 107  // 115200 baud at 100 MHz

volatile char rx_buffer[256];
volatile uint8_t rx_head = 0;
volatile uint8_t rx_tail = 0;

void external_irq_handler(void) {
    uint32_t src = kv_plic_claim();

    if (src == KV_PLIC_SRC_UART) {
        uint32_t status = kv_uart_irq_status();

        if (status & KV_UART_IE_RX_READY) {
            // Read all available characters
            while (kv_uart_rx_ready()) {
                rx_buffer[rx_head++] = kv_uart_getc();
            }
        }
    }

    kv_plic_complete(src);
}

int main(void) {
    // Initialize UART
    kv_uart_init(UART_BAUD_DIV);
    kv_uart_puts("UART Echo Demo\n");

    kv_plic_init_source(KV_PLIC_SRC_UART, 5);
    kv_uart_irq_enable(KV_UART_IE_RX_READY);
    kv_irq_enable();

    // Main loop: echo received characters
    while (1) {
        if (rx_head != rx_tail) {
            char c = rx_buffer[rx_tail++];
            kv_uart_putc(c);
        }
    }

    return 0;
}

=== Example 3: I2C EEPROM Access

[source,c]
----
#include "kv_platform.h"
#include "kv_i2c.h"

#define I2C_CLK_DIV 249  // 100 kHz at 100 MHz
#define EEPROM_ADDR 0x50

int eeprom_write(uint16_t mem_addr, const uint8_t *data, uint8_t len) {
    uint8_t buf[66];  // addr(2) + data(64 max)

    if (len > 64) return -1;

    buf[0] = (mem_addr >> 8) & 0xFF;
    buf[1] = mem_addr & 0xFF;
    for (uint8_t i = 0; i < len; i++) {
        buf[i + 2] = data[i];
    }

    return kv_i2c_master_write(EEPROM_ADDR, buf, len + 2);
}

int eeprom_read(uint16_t mem_addr, uint8_t *data, uint8_t len) {
    uint8_t addr_buf[2];
    addr_buf[0] = (mem_addr >> 8) & 0xFF;
    addr_buf[1] = mem_addr & 0xFF;

    // Write address
    kv_i2c_start();
    if (kv_i2c_write_byte((EEPROM_ADDR << 1) | 0) < 0) {
        kv_i2c_stop();
        return -1;
    }
    kv_i2c_write_byte(addr_buf[0]);
    kv_i2c_write_byte(addr_buf[1]);

    // Read data
    kv_i2c_start();  // Repeated START
    if (kv_i2c_write_byte((EEPROM_ADDR << 1) | 1) < 0) {
        kv_i2c_stop();
        return -1;
    }
    for (uint8_t i = 0; i < len; i++) {
        data[i] = kv_i2c_read_byte(i < len - 1);
    }
    kv_i2c_stop();

    return 0;
}

int main(void) {
    uint8_t write_data[] = {0xAA, 0xBB, 0xCC, 0xDD};
    uint8_t read_data[4];

    kv_i2c_init(I2C_CLK_DIV);

    // Write to EEPROM
    if (eeprom_write(0x0100, write_data, 4) < 0) {
        kv_magic_exit(1);
    }

    // Small delay for write cycle
    for (volatile int i = 0; i < 10000; i++);

    // Read back
    if (eeprom_read(0x0100, read_data, 4) < 0) {
        kv_magic_exit(2);
    }

    // Verify
    for (int i = 0; i < 4; i++) {
        if (read_data[i] != write_data[i]) {
            kv_magic_exit(3);
        }
    }

    kv_magic_exit(0);
    return 0;
}
----

=== Example 4: DMA Memory Copy with Performance Monitoring

[source,c]
----
#include "kv_platform.h"
#include "kv_dma.h"

#define DMA_CHANNEL 0
#define DATA_SIZE 4096

__attribute__((aligned(4))) uint8_t src_buffer[DATA_SIZE];
__attribute__((aligned(4))) uint8_t dst_buffer[DATA_SIZE];

int main(void) {
    // Initialize source buffer
    for (int i = 0; i < DATA_SIZE; i++) {
        src_buffer[i] = i & 0xFF;
    }

    // Reset and enable performance counters
    kv_dma_perf_reset();

    // Perform DMA copy using SDK (blocking 1D transfer)
    int ret = kv_dma_1d_copy(DMA_CHANNEL,
                             (uint32_t)src_buffer,
                             (uint32_t)dst_buffer,
                             DATA_SIZE);
    kv_dma_ch_reset(DMA_CHANNEL);  // Clear DONE flag

    if (ret != 0) {
        kv_magic_exit(1);  // AXI error
    }

    // Stop counting so results are stable
    kv_dma_perf_disable();

    // Read performance counters
    uint32_t cycles   = kv_dma_perf_get_cycles();
    uint32_t rd_bytes = kv_dma_perf_get_rd_bytes();
    uint32_t wr_bytes = kv_dma_perf_get_wr_bytes();

    printf("DMA: %u cycles, %u rd, %u wr\n", cycles, rd_bytes, wr_bytes);

    // Verify data
    for (int i = 0; i < DATA_SIZE; i++) {
        if (dst_buffer[i] != src_buffer[i]) {
            kv_magic_exit(2);
        }
    }

    kv_magic_exit(0);
    return 0;
}

== Appendix C: Performance Considerations

=== DMA vs CPU Copy

For bulk data transfers > 256 bytes, DMA provides significant performance benefits:

* *CPU Copy*: ~4-8 cycles per word (load + store + loop overhead)
* *DMA*: 1-2 cycles per word (overlapped read/write, no CPU involvement)

=== Cache Coherency

The KV32 SoC includes both an instruction cache (I-cache) and a data cache (D-cache). When using DMA, explicit cache maintenance is necessary to keep the caches coherent with memory:

* *Before a DMA read* (CPU writes data, DMA reads): flush the D-cache range so dirty lines are written back to SRAM before the DMA transfer begins.
+
[source,c]
----
kv_dcache_flush_range(buf, len);   // write-back dirty lines
// then program the DMA descriptor and start the transfer
----

* *After a DMA write* (DMA writes data, CPU reads): invalidate the D-cache range so the CPU fetches fresh data from SRAM rather than stale lines.
+
[source,c]
----
// wait for DMA transfer complete interrupt/poll
kv_dcache_inval_range(buf, len);   // discard stale cached lines
// now safe to read buf[]
----

* *Code loaded into SRAM by DMA* (e.g., overlay / firmware update): after invalidating the D-cache, also invalidate the I-cache so the CPU fetches the new instructions.
+
[source,c]
----
kv_dcache_inval_range(buf, len);
kv_fence_i();                      // synchronise I-cache
----

* Use `fence rw,rw` (or the `kv_fence()` helper) after MMIO writes to ensure store ordering before the DMA descriptor is submitted.

=== Interrupt Latency

Typical interrupt latency (cycles from assertion to handler entry):

* Minimum: ~15 cycles (no contention)
* Average: ~25 cycles (with PLIC arbitration)
* Maximum: ~100 cycles (worst-case pipeline state)

=== Scope

This appendix classifies SDK-visible features into three categories:

* Standard RISC-V ISA/privileged features

=== Classification Matrix

[cols="3,2,6"]
|===
| Feature | Category | Notes / Portability Impact

| RV32I base ISA
| Standard
| Fully standard base integer ISA behavior.

| M extension (multiply/divide)
| Standard optional
| Standard extension; portable to cores that implement `M`.

| A extension (atomics)
| Standard optional
| Standard extension; portable to cores that implement `A`.

| Zicsr CSR access
| Standard optional
| Standard extension used by `csr.h` and interrupt/exception control flows.

| Zifencei (`fence.i`)
| Standard optional

| Stack-guard custom CSRs (`sguard_base`, `spmin` at `0x7CC-0x7CD`)
| KV32-specific (non-standard)
| Hardware stack bound and low-water tracking exposed via `csr.h` (`read_csr_sguard_base`, `read_csr_spmin`).

| Stack-overflow exception (`KV_EXC_STACK_OVERFLOW`, cause `16`)
| KV32-specific (non-standard)
| Raised when stack pointer moves below configured guard; not a standard privileged-spec exception code.

| Trap-frame API (`kv_trap_frame_t`, frame-pointer exception handlers)
| KV32-specific SDK ABI
| Design choice for robust trap handling; not required by standard RISC-V ABI.

| Magic Device MMIO (`KV_MAGIC_BASE`)
| KV32-specific simulation interface
| Simulation/testbench-only host console+exit transport. Not synthesizable as a standard RISC-V device.

| DMA / Timer / WDT / GPIO / UART / SPI / I2C register maps
| KV32 platform-specific peripherals
| Device APIs are KV32 SoC specific even when functionally similar to common IP blocks.
|===

=== Important KV32 Notes

==== `fence.i` behavior

`kv_fence_i()` uses the standard `fence.i` instruction encoding, but KV32 documentation notes an additional implementation behavior: a full D-cache write-back may be performed to ensure self-modifying code visibility. Treat this as KV32-specific micro-architectural behavior, not a guaranteed generic RISC-V property.

==== PMA CSRs are custom

`kv_pma.h` exposes PMA control through custom M-mode CSRs in the `0x7C0-0x7CB` range. These CSRs are not part of the standard RISC-V privileged spec and must be abstracted out when porting firmware.

==== Stack-guard CSRs and exception are custom

`csr.h` exposes stack-guard controls through custom M-mode CSRs `0x7CC` (`SGUARD_BASE`) and `0x7CD` (`SPMIN`). KV32 also defines custom exception cause `16` (`KV_EXC_STACK_OVERFLOW`). These are KV32 architectural extensions and are not represented by standard `misa` extension bits.

==== Trap redirection model

Exception handlers should update `frame->mepc` in `kv_trap_frame_t` rather than directly writing `mepc` CSR. This is the KV32 SDK contract and ensures compatibility with the trap epilogue restore sequence.

=== Portability Checklist

When porting KV32 firmware to another RISC-V target:

1. Replace or abstract `kv_pma_*` APIs.
2. Replace Magic Device calls (`kv_magic_putc`, `kv_magic_exit`) with target-specific transport.
3. Validate availability of `Zicbom` and `Zifencei` before using cache maintenance helpers.
4. Treat trap frame layout and handler ABI as platform-specific unless explicitly reproduced.
5. Re-map all peripheral drivers to target MMIO register maps and interrupt IDs.
6. Replace or abstract stack-guard controls (`read_csr_sguard_base`, `read_csr_spmin`) and exception cause `16` handling.
